El experimento se basa en medir el rendimiento del algoritmo clásico de multiplicación de matrices utilizando distintas técnicas de paralelismo (OpenMP, POSIX y Fork) en varios tamaños de matriz y números de hilos. Para replicarlo, basta con descomprimir los archivos fuente, compilar con el comando make y ejecutar cada programa con el formato ./ejecutable tamMatriz numHilos, donde se pueden usar valores como 64, 128, 256, 512 y 1024 para el tamaño, y 1, 2, 4 u 8 para el número de hilos. Cada ejecución muestra el tiempo en milisegundos que tarda en completarse la operación, lo que permite comparar el comportamiento del algoritmo en modo secuencial y paralelo.

El proceso completo de evaluación puede automatizarse ejecutando el script lanzador.pl, que realiza múltiples repeticiones para cada combinación y guarda los resultados en archivos de datos (.dat) listos para exportar a una hoja de cálculo. Desde allí se pueden calcular los promedios, el speedup y la eficiencia, además de generar las gráficas de tiempo y rendimiento que reflejan el impacto del paralelismo sobre diferentes configuraciones de hardware y carga computacional.
